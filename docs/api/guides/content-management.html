Content Management Guide

Learn how to create, manage, and publish content using the Ultimate Marketing Team API.

Back to Documentation Home

Overview

The Content API allows you to manage various types of marketing content, including articles, social media posts, emails, ads, and landing pages. This guide covers common content management workflows and best practices.

Note: All content endpoints require authentication. Make sure to include your JWT token in the Authorization header.

Content Types

The API supports the following content types:

Type Description Use Cases
article Long-form written content Blog posts, knowledge base articles, tutorials
social_post Short-form content for social media Twitter/X posts, LinkedIn updates, Facebook posts
email Email marketing content Newsletters, promotional emails, transactional emails
ad Advertising content Banner ads, search ads, social media ads
landing_page Landing page content Campaign landing pages, product pages

Content Status Workflow

Content in the system follows a defined workflow with the following statuses:

  1. draft: Initial content creation (default)
  2. review: Content is ready for review
  3. approved: Content has been approved
  4. published: Content is live
  5. archived: Content is no longer active

To update a content item's status, use the update endpoint with the new status value.

Filter Content

Filter content items by status, type, or brand:

Endpoint: GET /api/v1/content?status=published&content_type=article&brand_id=123e4567-e89b-12d3-a456-426614174000 GET

Available filters:

  • status: Filter by content status (draft, review, approved, published, archived)
  • content_type: Filter by content type (article, social_post, email, ad, landing_page)
  • brand_id: Filter by brand ID
  • page: Page number for pagination (default: 1)
  • limit: Items per page (default: 20, max: 100)

Deleting Content

Delete Content

Remove a content item:

Endpoint: DELETE /api/v1/content/{content_id} DELETE

Response (204 No Content)

Warning: Deleting content is permanent and cannot be undone. Consider archiving content instead by updating its status to "archived".

Delete Image

Remove an image from content:

Endpoint: DELETE /api/v1/content/images/{image_id} DELETE

Response (204 No Content)

Code Examples

Python - Create and Publish Content

import requests
import json

# Authentication (replace with your token)
token = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
headers = {
    "Authorization": f"Bearer {token}",
    "Content-Type": "application/json"
}

# 1. Create content
content_data = {
    "title": "10 Tips for Marketing Success",
    "description": "Best practices for marketing in 2025",
    "content_type": "article",
    "body": "# 10 Tips for Marketing Success\n\n## Tip 1: Know Your Audience\n...",
    "status": "draft",
    "brand_id": "123e4567-e89b-12d3-a456-426614174000",
    "project_id": "123e4567-e89b-12d3-a456-426614174000",
    "metadata": {
        "target_audience": "Marketing professionals",
        "platforms": ["blog", "newsletter"]
    }
}

create_response = requests.post(
    "https://api.ultimatemarketingteam.com/api/v1/content",
    headers=headers,
    json=content_data
)

content = create_response.json()
content_id = content["id"]
print(f"Created content with ID: {content_id}")

# 2. Upload an image
image_path = "marketing-tips.jpg"
with open(image_path, "rb") as image_file:
    files = {"file": (image_path, image_file, "image/jpeg")}
    data = {
        "content_id": content_id,
        "brand_id": "123e4567-e89b-12d3-a456-426614174000"
    }
    image_response = requests.post(
        "https://api.ultimatemarketingteam.com/api/v1/content/upload-image",
        headers={"Authorization": f"Bearer {token}"},
        files=files,
        data=data
    )

image = image_response.json()
print(f"Uploaded image: {image['url']}")

# 3. Update content status to published
update_data = {
    "status": "published",
    "publish_date": "2023-11-15T09:00:00Z"
}

update_response = requests.put(
    f"https://api.ultimatemarketingteam.com/api/v1/content/{content_id}",
    headers=headers,
    json=update_data
)

updated_content = update_response.json()
print(f"Content published, status: {updated_content['status']}")

JavaScript - List and Filter Content

// Authentication (replace with your token)
const token = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";
const headers = {
  "Authorization": `Bearer ${token}`,
  "Content-Type": "application/json"
};

// Fetch content items for a specific brand and status
async function fetchBrandContent(brandId, status) {
  try {
    const response = await fetch(`https://api.ultimatemarketingteam.com/api/v1/content?brand_id=${brandId}&status=${status}`, {
      method: "GET",
      headers: headers
    });
    
    if (!response.ok) {
      throw new Error(`API error: ${response.status}`);
    }
    
    const data = await response.json();
    return data.items;
  } catch (error) {
    console.error("Error fetching content:", error);
    return [];
  }
}

// Display content items in the UI
async function displayPublishedContent() {
  const brandId = "123e4567-e89b-12d3-a456-426614174000";
  const publishedContent = await fetchBrandContent(brandId, "published");
  
  console.log(`Found ${publishedContent.length} published content items`);
  
  publishedContent.forEach(content => {
    console.log(`- ${content.title} (${content.content_type})`);
  });
}

displayPublishedContent();

Best Practices

  • Use descriptive titles and appropriate content types for better organization.
  • Include detailed metadata to make content more searchable and enable advanced filtering.
  • Follow the status workflow to track content through its lifecycle.
  • Set proper focal points on images for responsive display across different devices.
  • Use pagination parameters when dealing with large content collections.
  • Consider archiving content instead of deleting it to maintain a historical record.
  • For large images, compress them before uploading to improve performance.
  • Use Markdown formatting in the body field for rich content presentation.
Tip: When creating content programmatically, consider using a content approval workflow where items are first created as drafts, then moved to review and published states after appropriate checks.
